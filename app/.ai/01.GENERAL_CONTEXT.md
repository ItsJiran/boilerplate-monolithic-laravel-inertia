Enterprise Hybrid Monolith: Architecture & Reference Guide

Dokumen ini adalah Satu-satunya Sumber Kebenaran (Single Source of Truth) untuk standar penulisan kode dalam proyek ini.

> [!CAUTION]
> **CONTAINERIZED ENVIRONMENT ONLY**
> Jangan pernah menjalankan `php artisan migrate:fresh`, `php artisan db:seed`, atau perintah destruktif database lainnya secara langsung di host machine.
> Kita menggunakan environment Docker container. Semua operasi database harus dijalankan di dalam container atau dibiarkan ditangani oleh mekanisme container startup.

Fokus arsitektur ini adalah "Write Once, Serve Both". Satu Controller dan satu Service harus bisa melayani Frontend (Inertia.js) dan External API (Mobile App/Integration) secara bersamaan tanpa duplikasi logika.
1. Core Philosophy (Hukum Besi)

    Controller Agnostik:

        Controller tidak boleh peduli apakah kliennya adalah Browser (Inertia) atau Mobile App (JSON).

        Controller mendelegasikan keputusan output ke AppResponse helper.

    Service Wajib Stateless:

        Service hanya mengembalikan data (Model/Collection/Boolean). Service TIDAK BOLEH mengembalikan Inertia::render atau response()->json().

    Strict DTO:

        Input ke Service wajib menggunakan DTO.

    Smart Response Handling:

        Gunakan helper AppResponse untuk menangani output. Helper ini yang akan mendeteksi header request dan memberikan respon yang sesuai (JSON vs Inertia).

3. Reference Implementation (Study Case: Create Project)

Studi kasus: User membuat project baru. Jika lewat Web, redirect ke halaman index dengan flash message. Jika lewat API, return JSON data project.
A. The Data Transfer Object (DTO)

Sama seperti sebelumnya, tidak berubah.
PHP

namespace App\DTO\Project;

use Illuminate\Http\Request;
use Carbon\Carbon;

readonly class CreateProjectDto
{
    public function __construct(
        public int $userId,
        public string $name,
        public ?string $description,
        public Carbon $startDate
    ) {}

    public static function fromRequest(Request $request): self
    {
        $validated = (object) $request->validated();

        return new self(
            userId: $request->user()->id,
            name: $validated->name,
            description: $validated->description ?? null,
            startDate: Carbon::parse($validated->start_date)
        );
    }
}

B. The Service Layer

Sama seperti sebelumnya. Service tetap murni data.
PHP

namespace App\Services\Project;

use App\Models\Project;
use App\DTO\Project\CreateProjectDto;
use Illuminate\Support\Facades\DB;

class ProjectService
{
    public function createProject(CreateProjectDto $dto): Project
    {
        return DB::transaction(function () use ($dto) {
            return Project::create([
                'name' => $dto->name,
                'description' => $dto->description,
                'start_date' => $dto->startDate,
                'created_by' => $dto->userId,
                'status' => 'active'
            ]);
        });
    }
}

C. The Hybrid Response Helper (The Magic)

Lokasi: app/Services/Shared/AppResponse.php

Helper ini mendeteksi wantsJson().
PHP

namespace App\Services\Shared;

use Illuminate\Http\JsonResponse;
use Illuminate\Http\RedirectResponse;
use Inertia\Inertia;
use Inertia\Response as InertiaResponse;

class AppResponse
{
    /**
     * Handle sukses respon.
     * - API: Return JSON data.
     * - Web: Return Inertia Render ATAU Redirect dengan Flash Message.
     */
    public static function success(
        mixed $data = null, 
        string $message = 'Success', 
        ?string $view = null,      // Jika ingin render halaman (Inertia)
        ?string $route = null      // Jika ingin redirect (Inertia)
    ): JsonResponse|InertiaResponse|RedirectResponse
    {
        // 1. Jika Request adalah API (Mobile/Postman)
        if (request()->wantsJson()) {
            return response()->json([
                'success' => true,
                'message' => $message,
                'data'    => $data,
            ], 200);
        }

        // 2. Jika Request adalah Web (Inertia)
        
        // Kasus A: Redirect (Biasanya setelah Create/Update/Delete)
        if ($route) {
            return to_route($route)->with('success', $message);
        }

        // Kasus B: Render Halaman (Biasanya Get/Index/Show)
        if ($view) {
            // Kita bisa attach extra data (flash message) jika perlu
            return Inertia::render($view, $data);
        }

        // Default fallback (jarang terjadi)
        return back()->with('success', $message);
    }

    /**
     * Handle error respon.
     */
    public static function error(
        string $message = 'Error', 
        mixed $details = null, 
        int $status = 500
    ): JsonResponse|RedirectResponse
    {
        // 1. API: Return JSON Error
        if (request()->wantsJson()) {
            return response()->json([
                'success' => false,
                'message' => $message,
                'error_code' => $details,
            ], $status);
        }

        // 2. Web: Redirect Back dengan Flash Error
        return back()->withErrors(['message' => $message]);
    }
}

D. The Hybrid Controller

Lokasi: app/Http/Controllers/ProjectController.php

Perhatikan betapa bersihnya controller ini. Dia melayani dua "Tuan" (API & Web) sekaligus.
PHP

namespace App\Http\Controllers;

use App\Http\Controllers\Controller;
use App\Http\Requests\StoreProjectRequest;
use App\Services\Project\ProjectService;
use App\DTO\Project\CreateProjectDto;
use App\Services\Shared\AppResponse;

class ProjectController extends Controller
{
    public function __construct(
        protected ProjectService $projectService
    ) {}

    // 1. Contoh Read Data (Index)
    public function index()
    {
        // Misal service punya method getAll()
        $projects = $this->projectService->getAllProjects();

        // Hybrid Return:
        // - API: JSON list project
        // - Web: Render halaman 'Projects/Index' dengan props 'projects'
        return AppResponse::success(
            data: ['projects' => $projects],
            view: 'Projects/Index'
        );
    }

    // 2. Contoh Write Data (Store)
    public function store(StoreProjectRequest $request)
    {
        try {
            $dto = CreateProjectDto::fromRequest($request);
            $project = $this->projectService->createProject($dto);

            // Hybrid Return:
            // - API: JSON project baru + message
            // - Web: Redirect ke route 'projects.index' + Flash message
            return AppResponse::success(
                data: $project,
                message: 'Project created successfully',
                route: 'projects.index' 
            );

        } catch (\Exception $e) {
            return AppResponse::error(
                message: 'Failed to create project',
                details: $e->getMessage()
            );
        }
    }
}

4. Workflow Decision Tree

Saat membuat Controller method, gunakan panduan ini untuk parameter AppResponse::success:

    Apakah ini halaman GET (Menampilkan data)?

        Gunakan param view: 'Nama/Komponen/Inertia'.

        Isi data dengan array props yang dibutuhkan Vue/React.

        Hasil: API dapat JSON, Web dapat Halaman.

    Apakah ini aksi POST/PUT/DELETE (Modifikasi data)?

        Gunakan param route: 'nama.route.tujuan'.

        Isi message untuk notifikasi sukses.

        Hasil: API dapat JSON konfirmasi, Web melakukan Redirect + Flash Message.

    Apakah terjadi Error?

        Gunakan AppResponse::error.

        Hasil: API dapat status 500/400, Web akan redirect back dengan error bag (bisa ditampilkan di UI).

5. Frontend Guidelines (Inertia.js)

    Structure & State Management:
        - Gunakan `useForm` untuk semua state yang akan dikirim ke backend.
        - Hindari `useState` terpisah untuk input form (seperti dynamic inputs). Masukkan langsung ke dalam `useForm` data (array/object).
        - Update state menggunakan `setData`.

    Form Submission:
        - JANGAN PERNAH menyertakan properti `data` manual di argumen kedua `post/put/patch`. 
        - Inertia secara otomatis mengirimkan object `data` dari `useForm`.
        - Salah: `post(url, { data: { ... } })`
        - Benar: `post(url)` (Pastikan `data` sudah terupdate via `setData` sebelumnya logic state sudah ada di `data`).

6. Enum Usage & Storage Pattern

    Database Storage:
        - Enum SELALU disimpan sebagai STRING di database (kolom VARCHAR/TEXT).
        - JANGAN gunakan casting enum di Model (`protected $casts`).
        - Alasan: Jika enum berubah di masa depan (nilai atau case dihapus/diubah), data lama tidak akan error/corrupt.

    Controller & Service:
        - Gunakan Enum untuk type safety dan validasi di Controller dan Service layer.
        - Contoh: `UserRole::ADMIN->value`, `ProjectStatus::ACTIVE->value`.

    Model Methods:
        - Method di Model tetap menerima dan mengembalikan STRING.
        - Jika perlu konversi ke Enum, lakukan secara eksplisit di Controller/Service.
        
    Contoh Pattern:
        ```php
        // ❌ SALAH - Jangan cast di Model
        protected $casts = [
            'status' => ProjectStatus::class,
        ];
        
        // ✅ BENAR - Store as string
        $project->status = ProjectStatus::ACTIVE->value; // 'active'
        
        // ✅ BENAR - Convert explicitly when needed
        $enumValue = ProjectStatus::from($project->status);
        ```
