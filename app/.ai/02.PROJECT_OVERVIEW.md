# Project Overview & Tech Stack

## 1. Project Description
**Laravel Boilerplate** is an Enterprise Hybrid Monolith application designed as a robust starting point for modern web applications. It serves as a foundation for Multi-tenant SaaS platforms, enabling organizations to manage tenants and users efficiently.

**Key Features:**
- **Multi-tenancy:** Support for multiple tenants with isolated data/access.
- **User Management (RBAC):** Comprehensive Role-Based Access Control (Admin, User, Global Admin).
- **Data Import:** Excel import functionality for bulk data.
- **Location Intelligence:** Integrated reference data for Provinces, Regencies, Districts, and Villages.
- **Notifications:** Real-time notification system.

## 2. Technology Stack

### Backend (Laravel)
- **Framework:** Laravel 12.x
- **Language:** PHP ^8.2
- **Architecture:** Hybrid Monolith (Serving both Inertia.js and API consumers from single controllers).
- **Authentication:** Laravel Sanctum
- **Realtime:** Laravel Reverb / Pusher PHP Server
- **Excel Processing:** Maatwebsite Excel
- **API/Routing:** Hybrid routing (Web & API routes in `routes/web.php` and `api` prefixed routes).

### Frontend (Inertia.js)
- **Framework:** React 18.x
- **Glue:** Inertia.js 2.0
- **Build Tool:** Vite 7.x
- **Styling:** TailwindCSS 4.0, DaisyUI 5.x, Headless UI
- **State Management:** Zustand
- **Charting:** Tremor, Chart.js, React-Chartjs-2
- **HTTP Client:** Axios

### Database & Storage
- **Database:** SQLite (Default/Dev), likely MySQL/PostgreSQL for Prod.
- **Storage:** Local / AWS S3 (via Flysystem)

## 3. Key Modules & Domain Models

### Admin & Tenancy
- **Tenants:** `App\Models\Tenant` - Core entity for multi-tenancy.
- **Users:** `App\Models\User` - System users.
- **Roles:** `App\Models\Role`, `App\Models\UserRole` - Authorization & Permissions.

### Core Infrastructure
- **References:** `ReferenceProvince`, `ReferenceRegency`, `ReferenceDistrict`, `ReferenceVillage` - Regional data.
- **Notifications:** `App\Models\Notification`.

## 4. Development Standards
- Follows the **"Write Once, Serve Both"** philosophy (see `01.GENERAL_CONTEXT.md`).
- strict DTO usage for Services.
- `AppResponse` helper for unified response handling.
