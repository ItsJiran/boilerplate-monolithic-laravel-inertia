# Testing Standards & Patterns

This document describes the testing standards for this boilerplate application, specifically regarding the **Multi-tenancy** handling strategy and the use of **Test Helpers**.

## 1. Testing Philosophy

- **Feature Tests** are prioritized for controller endpoints (Web/API).
- **Unit Tests** are used for complex business logic in the Service layer.
- Each test must run in an isolated environment (`RefreshDatabase`).
- **Database Seeding** in tests uses a lightweight Helper Trait, **NOT** full `db:seed`, to maintain execution speed.

## 2. Helper `WithTestSeeder`

We have created a custom trait `Tests\Support\WithTestSeeder` to standardize the basic data setup required by almost all Multi-tenant features.

This trait provides modular methods to create:
1. **Roles**: Super Admin, Admin, User.
2. **Tenant Context**: New Tenant + Admin User assigned to that tenant.
3. **Reference Data**: Geographic data (Province/City/etc).

### How to Use

Add the `WithTestSeeder` trait to your Test class:

```php
namespace Tests\Feature;

use Tests\TestCase;
use Tests\Support\WithTestSeeder; // <--- Import Trait
use Illuminate\Foundation\Testing\RefreshDatabase;

class MyFeatureTest extends TestCase
{
    use RefreshDatabase;
    use WithTestSeeder; // <--- Use Trait

    protected function setUp(): void
    {
        parent::setUp();
        
        // 1. Setup Basic Environment (Roles, Tenant, Admin User)
        // This method will populate class properties: $this->tenant, $this->adminUser, $this->roleAdmin
        $this->seedTenantAndAdmin();
        
        // 2. Setup Additional Data (Optional)
        // $this->seedGeography();  // If region data is needed
        
        // 3. Set User Login Context
        $this->actingAs($this->adminUser);
        
        // 4. (Important) Clear RBAC/Permission Cache if any
        \Illuminate\Support\Facades\Cache::flush();
    }
```

## 3. Tenant Context Pattern

In this Multi-tenant application, many features require `tenant_id` either in the URL, Request Body, or query filter.

### Accessing Tenant Data
After running `$this->seedTenantAndAdmin()`, you can access the tenant object via `$this->tenant`.

Example usage in Request:

```php
public function test_can_view_dashboard()
{
    // Accessing route that requires tenant_id parameter
    $response = $this->get(route('dashboard', [
        'tenant_id' => $this->tenant->id 
    ]));
    
    $response->assertStatus(200);
}

public function test_can_store_data()
{
    $payload = [
        'tenant_id' => $this->tenant->id, // Tenant_id is mandatory
        'name' => 'Test Data',
    ];
    
    $this->post(route('data.store'), $payload)
         ->assertStatus(200);
}
```

### Available Properties
The `WithTestSeeder` helper provides quick access properties:

- `$this->tenant` (`Tenant` Model)
- `$this->adminUser` (`User` Model with Admin role in that tenant)
- `$this->roleAdmin`, `$this->roleSuperAdmin` (`Role` Models)

## 4. Multi-tenancy Isolation Testing (Mandatory)

It is crucial to ensure **User Tenant A cannot see Tenant B's data**.

### Isolation Testing Pattern

1.  Setup Tenant A (via `setUp`).
2.  Manually create Tenant B inside the test method.
3.  Create a resource belonging to Tenant B.
4.  Try to access Tenant B's resource using Tenant A's user.
5.  Assert `403 Forbidden` or `404 Not Found` (depending on policy implementation).

**Example Test Case:**

```php
public function test_cannot_access_other_tenant_customer()
{
    // 1. Tenant A (Current User)
    $myCustomer = Customer::factory()->create(['tenant_id' => $this->tenant->id]);

    // 2. Tenant B (Other Tenant)
    $otherTenant = Tenant::factory()->create();
    $otherCustomer = Customer::factory()->create(['tenant_id' => $otherTenant->id]);

    // 3. Attempt to view Tenant B's customer
    $response = $this->get(route('marketing.customers.show', [
        'customer' => $otherCustomer->id,
        'tenant_id' => $this->tenant->id // Passing my tenant ID, but resource is not mine
    ]));

    // 4. Assert Access Denied
    $response->assertStatus(403);
}

public function test_customer_list_is_scoped_to_tenant()
{
    // Create data for both tenants
    Customer::factory()->create(['tenant_id' => $this->tenant->id, 'contact_name' => 'My Data']);
    Customer::factory()->create(['tenant_id' => Tenant::factory()->create()->id, 'contact_name' => 'Other Data']);

    $response = $this->get(route('marketing.customers.index', ['tenant_id' => $this->tenant->id]));

    $response->assertSee('My Data');
    $response->assertDontSee('Other Data');
}
```

## 5. Common Troubleshooting

1. **Error 403 / Unauthorized**: 
   - Ensure `$this->actingAs($this->adminUser)` is called.
   - Ensure the user has a role in the correct tenant (`seedTenantAndAdmin` handles this automatically).
   
2. **Logic Error / Data Mismatches**:
   - Ensure the `tenant_id` sent in the request matches `$this->tenant->id`.
   - RBAC Cache might persist, make sure `Cache::flush()` is in `setUp`.

3. **Error CSRF**:
   - Untuk feature test yang mensimulasikan API call atau form submission, Anda bisa disable CSRF middleware::
     ```php
     $this->withoutMiddleware(\Illuminate\Foundation\Http\Middleware\ValidateCsrfToken::class);
     ```
