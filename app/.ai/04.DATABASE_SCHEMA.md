# Database Schema Documentation

This document provides a comprehensive overview of the database schema for the Boilerplate Application, based on the migration files located in `database/migrations`.

> [!CAUTION]
> **CONTAINERIZED ENVIRONMENT ONLY**
> Never run `php artisan migrate:fresh`, `php artisan db:seed`, or other destructive database commands directly on the host machine.
> We use a Docker container environment. 

> **Last Updated:** March 9, 2026 - Synchronized with migrations up to `2026_03_02_000006_drop_legacy_slug_registry_tables.php`

## Core System Tables

### `users`
- `id` (PK)
- `name` (string)
- `username` (string)
- `email` (string, unique)
- `email_verified_at` (timestamp, nullable)
- `password` (string)
- `profile_path` (string, nullable)
- `remember_token`
- `softDeletes`
- `timestamps`

### `roles`
- `id` (PK)
- `name` (string)
- `slug` (string)
- `description` (string, nullable)
- `timestamps`
- `softDeletes`

### `tenants`
- `id` (PK)
- `name` (string)
- `slug` (string)
- `profile_path` (string, nullable)
- `timestamps`
- `softDeletes`

### `user_roles`
- `id` (PK)
- `user_id` (FK -> users)
- `role_id` (FK -> roles)
- `tenant_id` (FK -> tenants, nullable)

- `timestamps`

## Customer Type Management Tables [Added in `2026_03_02`]

### `customer_types`
- `id` (PK)
- `tenant_id` (FK -> tenants, onDelete: cascade)
- `customer_kind` (string, 20) - Values: 'individual' | 'organization'
- `name` (string, 100)
- `slug_mode` (string, 20, default 'auto_generated') - Values: 'auto_generated' | 'manual_external'
- `slug_prefix` (string, 10, nullable) - Prefix for auto-generated slugs
- `slug_counter` (unsigned int, default 0) - Atomic counter for slug generation
- `slug_validation` (string, 255, nullable) - Regex validation for manual_external mode
- `slug_validation_hint` (string, 255, nullable) - Hint text for user input validation
- `is_locked` (boolean, default false) - Lock flag during migration or admin lock
- `is_active` (boolean, default true)
- `timestamps`
- `softDeletes`
- Unique Index: [`tenant_id`, `customer_kind`, `name`]
- Unique Index: [`tenant_id`, `customer_kind`, `slug_prefix`]

### `slug_migration_jobs`
- `id` (PK)
- `tenant_id` (unsigned bigint, index)
- `customer_type_id` (FK -> customer_types, onDelete: cascade, index)
- `old_slug_mode` (string, 32)
- `new_slug_mode` (string, 32)
- `old_slug_prefix` (string, 20, nullable)
- `new_slug_prefix` (string, 20, nullable)
- `old_slug_validation` (string, 255, nullable)
- `new_slug_validation` (string, 255, nullable)
- `status` (string, 32, default 'generating', index) - Values: generating | pending_approval | applying | completed | completed_with_errors | cancelled
- `total_items` (unsigned int, default 0)
- `applied_count` (unsigned int, default 0)
- `skipped_count` (unsigned int, default 0)
- `failed_count` (unsigned int, default 0)
- `created_by` (unsigned bigint, nullable)
- `approved_by` (unsigned bigint, nullable)
- `approved_at` (timestamp, nullable)
- `completed_at` (timestamp, nullable)
- `error_message` (text, nullable)
- `timestamps`

### `slug_migration_items`
- `id` (PK)
- `migration_job_id` (FK -> slug_migration_jobs, onDelete: cascade, index)
- `customer_id` (FK -> customers, onDelete: cascade, index)
- `current_slug` (string, 100, nullable) - Snapshot of slug when preview was created
- `proposed_slug` (string, 100, nullable) - Slug to be applied (auto-generated or manual)
- `status` (string, 32, default 'pending', index) - Values: pending | applied | skipped | failed
- `admin_note` (text, nullable)
- `error_message` (text, nullable)
- `applied_at` (timestamp, nullable)
- `timestamps`
- Unique Index: [`migration_job_id`, `customer_id`]

### `customer_slug_ledger`
- `id` (PK)
- `tenant_id` (unsigned bigint, index)
- `customer_id` (FK -> customers, onDelete: cascade, index)
- `customer_type_id` (unsigned bigint, index)
- `old_slug` (string, 100, nullable)
- `new_slug` (string, 100)
- `change_reason` (string, 32) - Values: initial | manual_edit | migration | admin_override
- `changed_by` (unsigned bigint, nullable) - User who made the change (null = system/job)
- `migration_item_id` (FK -> slug_migration_items, onDelete: set null, nullable, index)
- `created_at` (timestamp, useCurrent) - Append-only log
